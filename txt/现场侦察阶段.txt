# **“现场侦察阶段（灾情快速侦查）”需求规格说明书 V1.0**

> 适用范围：AI应急大脑与全空间智能车辆系统 · **前突侦察指挥车**
> 阶段边界：车辆抵达灾区外缘/首个安全驻扎点起，至“首轮全域（或重点区）侦察完成并形成可执行救援方案与上报材料”止。
> 能力底座：空中/地面侦察智能体、实时风险评估、救援方案生成、任务智能分解/分发、ASR/TTS、多机协同、通信协调、态势自动标绘等。

> 注：你文中“**酒宴**”应为“**救援**”之笔误；本文按“救援”处理，避免歧义。

---

## 1. 阶段目标（3×3 法则）

* **看清楚**：在 30–60 分钟内拿到“可落地”的灾情真相图（道路通行性、受损设施、人员密集/困守点、次生灾害风险区）。
* **保安全**：前突车队与后续救援力量的**行进与驻扎安全**（道路、气象/地质、危化）。
* **能落地**：基于现有装备与已接入的社会/专业救援力量，生成**首轮救援方案**并下发任务；同时形成**高层态势“建议件”**（帮助上级统筹大兵团救援）。

---

## 2. 角色与席位（前突态 · 席位职责补强）

* **指挥席（C2）**：对上执行指挥大厅命令（联网时），对内拍板区域划分、优先序、风险容忍度与资源投向；审批“高层态势建议件”。
* **驾驶席（行进安全）**：保障车辆机动，执行道路风险提醒与避险；语音问答“前方风险/最近安全点”。
* **侦察席（空地情报）**：操控“神眼”与强声；调度本车/编队无人机，负责自动+手动标注与成果审核。
* **协调席（对外协同）**：拉通医院/消防/武警/社会救援队 APP 接入与手录；汇总口头情报→地图标绘→提交变更建议。
* **无人机操作席（分布在携机车辆）**：各车各管一片，按扇区/航线模板起降，执行 YOLO 初筛 + 多模态大模型复核的人在回路（HITL）。

---

## 3. 现场侦察业务流程（泳道/时序）

1. **抵达与“智能选址”建点**

   * AI在 90 秒内基于 DEM/地质/风场/退路/通信覆盖，推荐 **3 个安全驻扎点（FOB）**，指挥席择一落地；同时给出**备用撤离线**与**禁入区**。
2. **区域切片与任务扇区化**

   * 将任务区划分为若干扇区（医院/学校/加油站/化工厂/桥隧等重点要素优先），为每个扇区生成**空地协同侦察任务卡**与“一键起飞”模板（高度/航线/载荷/回传频率）。
3. **无人机快速侦察闭环**

   * **YOLO 初筛** → **多模态大模型复核** → **侦察席/操作席确认** → 自动上图（置信度/证据链），关键目标（伤员聚集、道路阻断、烟雾/泄漏）T+秒级上屏。
4. **地面快速勘察与先期救助**

   * 车队分散小组实施：道路可通性验证、群众转移引导、医疗急救（简易分级）、临时安置点踏勘与建立。
5. **安全护航与持续避险**

   * 对在途与作业小队提供**动态风险监护**（暴雨/滑坡/危化扩散/桥梁异常），达阈值自动强提醒并**建议换线/临停/改驻点**。
6. **方案生成与任务下发**

   * AI基于**侦察成果+现有装备+APP接入力量**生成救援方案（可编辑），指挥席审定后由**任务智能分发**推送到队伍端 APP 并在 3D 地图实时跟踪执行态。
7. **对上沟通：高层态势“建议件”**

   * 若出现**化工泄漏/大桥中断/堰塞风险/火场失控**等超出前突能力的情形，AI与指挥席合成**省级视角态势建议件**（见 §6），以建议而非命令的姿态报送指挥大厅。

## 4. 功能需求（FR）

### FR-01 指挥所智能选址（FOB）

* **输入**：DEM、地质敏感分区、风向/降水、临近要素（学校/医院/化工/变电站）、通信覆盖、撤离线/补给点。
* **输出**：3 个候选（安全评分、可落点、回转空间、LoS 视距、通信链路），并生成**警戒半径与禁入扇区**。
* **时限**：≤90s；**一键落地**生成“驻扎点任务卡”（警戒/交通引导/起降区）。

### FR-02 扇区化侦察与一键起飞

* **自动分区**：按地物密度、要素优先级与空域约束生成侦察扇区。
* **任务卡**：航线、传感器（可见光/红外/气体/扩展载荷）、高度/速度、回传码率、返航点、禁飞区。
* **一键化**：席位上 **“一键起飞”**、**“一键复飞”**、**“一键补侦”** 三键。

### FR-03 YOLO 初筛 + 多模态复核 + 人在回路（HITL）

* **流程**：

  1. 模型对视频帧/瓦片做**目标初筛**（人/阻断/火点/烟雾/油污/化学云）；
  2. 多模态大模型进行**语义验证**与**上下文判别**（如“疑似氯气云”“桥面拉裂”）；
  3. 侦察席/操作席**确认/驳回/标注补充**；
  4. 结构化写入“发现表（Findings）”，并自动叠加到 3D 地图。
* **约束**：所有“红色风险”必须二次确认；未确认项以“低置信度”呈现。

### FR-04 地面先期救助（轻量化）

* **能力**：人员转移组织、简易医疗（止血/包扎/固定/基础药品）、临时安置（搭设/指引）、危险区隔离（强声+标识）。
* **AI 辅助**：按**优先级**提示“最近可用安置点/医疗点/补给点”，并输出“到点清单”。

### FR-05 任务生成 → 智能分解 → 智能分发

* **生成**：救援方案生成模型基于**任务目标-可用力量-约束**出多方案比选（时效/安全/资源占用）。
* **分解**：任务智能分解为岗位级**任务卡**（地点/时窗/物资/到点清单/风险条款）。
* **分发**：任务智能分发推送到队伍 APP（含定位/轨迹采集/回执），并在 3D 地图显示**任务与队伍运动轨迹**。

### FR-06 3D 地图“所见即战场”

* **联动图层**：扇区/航线、发现（人/阻断/危化/火点等）、社会力量与救护接驳点、任务与队伍轨迹、驻扎点与禁入区。
* **实时性**：发现与任务状态 T+秒级上图；离线缓存与恢复后自动补齐。

### FR-07 安全护航（队伍保护）

* **监护对象**：前突车队与分散作业小队（含携机车）。
* **机制**：实时风险评估 + 预警监测，对**暴雨/滑坡/洪水/危化扩散**触发红/橙/黄阈值；**红/橙**触发强提醒与**替代路线/改驻点**建议。

### FR-08 一键宏操作与语音交互

* **一键宏**：

  * **一键“起点建点”**（FOB 选址→落地→周界布控）
  * **一键“全域首侦”**（扇区化→多机起飞→自动回传→成果上图）
  * **一键“上报建议件”**（见 FR-10）
* **语音**：驾驶/侦察/指挥均可口令式交互（查询前方风险、最近安全点、扇区完成度、建议方案摘要）。

### FR-09 联网/断网双态作业

* **联网时**：严格执行指挥大厅指令流；本地方案为**备选建议**，需上级确认后下发执行。
* **断网时**：本地自治闭环（决策与留痕）；复联后**差异合并与审计回放**。

### FR-10 高层态势“建议件”（SitRep-S, 面向省级）

* **定位**：**建议**而非命令；内容供上级统筹大型救援使用。
* **结构**：

  1. **总体态势图**（3D 概览：断点/危化/水势/火势/人群/队伍分布）；
  2. **关键阻断与恢复建议**（如“XX桥断：需桥梁抢修队×2、总计×人、设备×台、估计×小时，打通×号走廊”）；
  3. **危化处置建议**（“XX 化工泄漏：建议防化分队×支，携×类侦检/洗消装备，警戒半径×km，前突已封控×km²”）；
  4. **增援与阶段目标**（优先级、到达窗口、接驳点/集结点）；
  5. **风险展望**（余震/雨情/次生灾害时间窗）。
* **生产**：指挥席口述/补充，AI自动成文与制图；一键导出 PDF/大屏。

## 5. 数据模型（核心草案）

**SafeBase（驻扎点）**

```json
{"id":"SB-...","point":[...],"risk":{"slope":0.12,"flood":0.05,"chem":0.00},
 "los_score":0.86,"landing_ok":true,"egress":["R1","R2"],"comm":"SAT|RELAY","status":"ACTIVE"}
```

**ReconSector（扇区）**

```json
{"id":"SEC-...","geom":{"type":"Polygon","coordinates":[...]},
 "priority":"HIGH|MED|LOW","targets":["HOSPITAL","SCHOOL","BRIDGE","CHEM_PLANT"],
 "uav_plan":{"alt_m":120,"payloads":["EO","IR","GAS"],"rtk":true}}
```

**Finding（侦察发现）**

```json
{"id":"F-...","type":"ROAD_BLOCK|CASUALTY_GROUP|CHEM_CLOUD|BRIDGE_DAMAGE",
 "source":"UAV|GROUND|SOCIAL","confidence":0.82,
 "evidence":["vid://...","img://..."],"geom":{"type":"Point","coordinates":[...]},
 "status":"PENDING|CONFIRMED|REJECTED"}
```

**Mission（任务）**

```json
{"id":"MIS-...","goal":"EVACUATE|MEDICAL|CLEAR_ROAD|SUPPLY_DROP",
 "sector":"SEC-...","team":"UNIT-...","checklist":["PPE","SATCOM","STRETCHER"],
 "window":{"start":"...","end":"..."}, "risk_clause":["ORANGE_SLOPE","FLOOD_WARN"]}
```

**SitRepS（高层建议件）**

```json
{"id":"SR-...","scope":"PROVINCE","summary":"XX桥断，需抢修××",
 "recommendations":[{"type":"BRIDGE_REPAIR","units":2,"people":38,"equip":["100t_crane","bailey_bridge"]}],
 "exclusion":{"chem_radius_km":2.5},"map":"map://overview-3d"}
```

## 6. UI/UE 关键点（席位化最小操作）

* **指挥席**：

  * 顶部“**一键**”：**建点** / **首侦** / **建议件** / **下发**；
  * 三栏：扇区进度条、关键风险卡、建议方案卡（可编辑/审定）。
* **侦察席/操作席**：

  * 双窗：神眼画面 + 地图；左侧“发现流”，右侧“确认/标注/派发”；
  * 三键：**起飞** / **复飞** / **补侦**。
* **协调席**：

  * APP 邀请码、队伍清单（在线/离线）、手动标绘提案表单与证据上传；
  * 一键“接驳点广播”。
* **驾驶席**：

  * HUD 风险条 + 语音问答 + 最近安全点卡片。
* **公共**：

  * 3D 地图“任务与队伍轨迹”随动显示；强提醒 10 秒不可关闭；**离线模式**明显标识。

## 7. 接口（API/消息）与离线策略

* `POST /fob/recommend` → 返回 3 个安全驻扎点与评分。
* `POST /sectors/auto_partition` → 输入范围与优先要素 → 扇区集。
* `POST /uav/mission/quickstart` → 一键起飞（支持模板）。
* `POST /findings/ingest` → YOLO/多模态复核结果入库（含证据/置信度）。
* `POST /plan/generate` → 生成救援方案（多方案比选+解释）。
* `POST /tasks/dispatch` → 方案下发为任务卡，推送到 APP 并返回回执/轨迹流。
* `POST /sitreps/generate?type=S` → 高层建议件（PDF/大屏/地图书签）。
* **离线**：所有关键写操作本地 WAL 排队，复联后幂等回放；冲突按“版本时间线+席位优先级”合并。

## 8. 非功能性指标

* **时效**：FOB 推荐 ≤90s；扇区与航线生成 ≤60s；YOLO 初筛 ≤1s/帧；多模态复核 ≤3s/事件；一键方案 ≤10s；建议件生成 ≤20s。
* **可用性**：≥99.9%；断网可运行；多网自适应（卫星/5G/专网/中继）与带宽分级（指令>告警>低码视频>日志）。
* **安全**：分级分域、二次认证、端到端加密、全链路审计。

## 9. 验收与测试用例（节选）

1. **FOB 选址**：在山地临河场景给出 3 候选；选中后自动生成周界与起降区，误报/漏报隐患率 ≤5%。
2. **首轮侦察**：10 km² 扇区化 + 三机并行起飞，30 分钟内“道路可通性+伤员聚集+危化迹象”全部上图。
3. **HITL 流**：YOLO 判为“疑似泄漏云”→ 多模态复核为“否”→ 人工确认“否”，最终标注状态正确且可追溯。
4. **先期救助**：模拟 20 名伤者，AI 推荐安置点与接驳路线，一线任务卡到达 APP 并有轨迹回执。
5. **安全护航**：短临暴雨触发“橙色”，系统给出“改驻点+补侦”建议；驾驶/侦察席收到强提醒并确认。
6. **建议件**：化工厂泄漏与桥断并存，AI 生成省级建议件含队伍/人数/设备清单与态势图，导出 PDF 与大屏。
7. **断网合并**：离线 20 分钟作业，复联后 60 秒内完成差异合并，地图时间线与审计链完整。

## 10. 里程碑（建议）

* **M1（2 周）**：FOB 选址 + 扇区化 + 一键起飞 + 发现上图（基础版）。
* **M2（4 周）**：YOLO+多模态复核 + HITL + 任务分解与 APP 下发 + 轨迹回执。
* **M3（6 周）**：安全护航（红橙黄阈值）+ 一键建议件 + 断网差异合并。
* **M4（8 周）**：性能/可靠性压测、全席位联动演练与复盘。

### 实施提示（给研发与产品）

* 先把“**一键**三件套”（建点/首侦/建议件）磨到极致顺滑，再叠更多炫技功能；
* 把“**HITL**闭环”当作质量闸门：任何红色风险都要 AI→AI 复核→人确认三道关；
* **地图即工单**：任务=地图要素+时间窗+风险条款，所有变更都在地图上完成；
* **建议件与命令分离**：前突只给“建议”，服从上级统一调度，合规且不越位。
