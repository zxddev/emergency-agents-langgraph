# 意图：evidence_bookmark_playback

## 标准槽位
- `target_type`
- `target_id`
- `action`
- 可选：`duration_sec`、`uri`

## 当前实现
- 仅定义 Schema，缺少实际处理逻辑。

## 待完成闭环
1. **明确动作语义**：确认 action 枚举（创建书签、回放片段等）及对应服务。
2. **实现 Handler**：对接证据管理系统或视频平台，执行书签/回放操作。
3. **返回结果**：提供执行状态、跳转链接、失败原因等信息。
4. **权限与审计**：记录操作人员、时间、证据来源。

## 依赖与注意事项
- 需与证据系统团队对齐 API 设计与安全策略。
- 回放可能涉及大流量，注意限流与缓存策略。
